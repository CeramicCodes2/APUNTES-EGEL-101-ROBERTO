# [[3.6.13. Python (7 de julio de 2022). Comparing Python to other languages. Python|Python - Comparing Python to other languages]]
Python se compara a menudo con otros lenguajes interpretados, como [[3.1.1. Java|Java]], JavaScript, Perl, Tcl o Smalltalk. También se pueden hacer comparaciones con [[3.1.7. C++|C++]], Common Lisp y Scheme, lo que puede resultar esclarecedor. En esta sección, compararé brevemente Python con cada uno de estos lenguajes. Estas comparaciones se centran únicamente en cuestiones del lenguaje. En la práctica, la elección de un lenguaje de programación a menudo está dictada por otras restricciones del mundo real, como costos, disponibilidad, formación, inversión previa o incluso apego emocional. Dado que estos aspectos son altamente variables, parece una pérdida de tiempo considerarlos en gran medida en esta comparación.

## [[3.1.1. Java|Java]]
En general, ==se espera que los programas de Python se ejecuten más lentamente que los programas de Java, pero también requieren mucho menos tiempo para desarrollarse==. Los programas de Python suelen ser de ==3 a 5 veces más cortos que los programas de Java equivalentes==. Esta diferencia se debe a los tipos de datos de alto nivel incorporados en Python y a su tipado dinámico. Por ejemplo, ==un programador de Python no pierde tiempo declarando los tipos de argumentos o variables==, y los tipos polimórficos poderosos de lista y diccionario de Python, para los cuales se incorpora un sólido soporte sintáctico directamente en el lenguaje, se utilizan en casi todos los programas de Python. ==Debido al tipado en tiempo de ejecución, el tiempo de ejecución de Python debe trabajar más que el de Java==. Por ejemplo, al evaluar la expresión a+b, debe inspeccionar primero los objetos a y b para averiguar su tipo, que no se conoce en tiempo de compilación. Luego invoca la operación de suma adecuada, que puede ser un método definido por el usuario sobrecargado. Java, por otro lado, puede realizar una suma eficiente de enteros o números de punto flotante, pero requiere declaraciones de variables para a y b y no permite la sobrecarga del operador + para instancias de clases definidas por el usuario.

Por estas razones, ==Python es mucho más adecuado como un "lenguaje de pegamento"==, mientras que ==Java se caracteriza mejor como un lenguaje de implementación de bajo nivel==. De hecho, ambos se complementan de manera excelente. Los componentes se pueden desarrollar en Java y combinarse para formar aplicaciones en Python; ==Python también se puede utilizar para prototipar componentes hasta que su diseño pueda "solidificarse" en una implementación de Java==. Para respaldar este tipo de desarrollo, se está desarrollando una implementación de Python escrita en Java, que permite llamar código de Python desde Java y viceversa. En esta implementación, el código fuente de Python se traduce a bytecode de Java (con la ayuda de una biblioteca en tiempo de ejecución para admitir la semántica dinámica de Python).

## JavaScript
==El subconjunto "basado en objetos" de Python es aproximadamente equivalente a JavaScript==. Al igual que JavaScript (y a diferencia de Java), Python admite un estilo de programación que utiliza funciones y variables simples sin recurrir a definiciones de clase. Sin embargo, para JavaScript, eso es todo lo que hay. Por otro lado, Python admite la escritura de programas mucho más grandes y una mejor reutilización de código a través de un verdadero estilo de programación orientada a objetos, donde las clases y la herencia desempeñan un papel importante.

## Perl
Python y Perl provienen de un fondo similar (secuencias de comandos de Unix, que ambos han superado con creces), y tienen muchas características similares, pero siguen una filosofía diferente. ==Perl enfatiza el soporte para tareas comunes orientadas a aplicaciones==, como tener expresiones regulares incorporadas, exploración de archivos y funciones de generación de informes. ==Python enfatiza el soporte para metodologías de programación comunes, como el diseño de estructuras de datos y la programación orientada a objetos, y alienta a los programadores a escribir un código legible (y, por lo tanto, mantenible)== proporcionando una notación elegante pero no demasiado críptica. Como consecuencia, Python se acerca a Perl pero rara vez lo supera en su dominio de aplicación original; sin embargo, Python tiene una aplicabilidad mucho más amplia que el nicho de Perl.

## TCL
==Al igual que Python, Tcl es utilizable como un lenguaje de extensión de aplicaciones, así como un lenguaje de programación independiente==. Sin embargo, Tcl, que tradicionalmente almacena todos los datos como cadenas, es deficiente en cuanto a estructuras de datos y ==ejecuta código típico mucho más lentamente que Python==. Tcl también carece de características necesarias para escribir programas grandes, como espacios de nombres modulares. Por lo tanto, mientras que una "aplicación grande" típica que utiliza Tcl generalmente contiene extensiones Tcl escritas en C o C++ que son específicas para esa aplicación, ==una aplicación de Python equivalente a menudo se puede escribir en "Python puro"==. Por supuesto, el desarrollo en Python puro es mucho más rápido que tener que escribir y depurar un componente en C o C++. Se ha dicho que la única cualidad redentora de Tcl es el kit de herramientas Tk. Python ha adoptado una interfaz a Tk como su biblioteca de componentes GUI estándar.

Tcl 8.0 aborda los problemas de velocidad al proporcionar un compilador de bytecode con soporte limitado de tipos de datos y agrega espacios de nombres. Sin embargo, sigue siendo un lenguaje de programación mucho más engorroso.

## Smalltalk
Quizás ==la mayor diferencia entre Python y Smalltalk es la sintaxis más "mainstream" de Python==, lo que le otorga una ventaja en la formación de programadores. Al igual que Smalltalk, Python tiene tipado y enlace dinámico, y todo en Python es un objeto. Sin embargo, Python distingue los tipos de objetos incorporados de las clases definidas por el usuario y actualmente no permite la herencia de tipos incorporados. La biblioteca estándar de tipos de datos de colección de Smalltalk está más pulida, mientras que la biblioteca de Python tiene más facilidades para tratar con realidades de Internet y la World Wide Web, como correo electrónico, HTML y FTP.

Python tiene una filosofía diferente con respecto al entorno de desarrollo y la distribución de código. Donde Smalltalk tradicionalmente tiene una "imagen del sistema" monolítica que comprende tanto el entorno como el programa del usuario, Python almacena tanto módulos estándar como módulos de usuario en archivos individuales que se pueden reorganizar o distribuir fácilmente fuera del sistema. Una consecuencia es que existen más de una opción para adjuntar una Interfaz Gráfica de Usuario (GUI) a un programa de Python, ya que la GUI no está incorporada en el sistema.

## [[3.1.7. C++|C++]]
Casi todo lo dicho sobre Java también se aplica a C++, pero de manera más pronunciada: ==cuando el código de Python suele ser de 3 a 5 veces más corto que el código de Java equivalente, a menudo es de 5 a 10 veces más corto que el código de C++==. La evidencia anecdótica sugiere que un programador de Python puede completar en dos meses lo que dos programadores de C++ no pueden completar en un año. ==Python brilla como un lenguaje de unión, utilizado para combinar componentes escritos en C++==.

## Common Lisp y Scheme
==Estos lenguajes son cercanos a Python en su semántica dinámica==, pero tan diferentes en su enfoque de la sintaxis que la comparación se convierte en casi una discusión religiosa: ¿es la falta de sintaxis en Lisp una ventaja o una desventaja? Debe tenerse en cuenta que Python tiene capacidades introspectivas similares a las de Lisp, y los programas de Python pueden construir y ejecutar fragmentos de programa sobre la marcha. Por lo general, las propiedades del mundo real son decisivas: Common Lisp es grande (en todos los sentidos), y el mundo Scheme está fragmentado entre muchas versiones incompatibles, mientras que Python tiene una única implementación gratuita y compacta.
